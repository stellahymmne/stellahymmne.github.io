Magical Language

I revise. I pace.

If you have seen something which requires an act of magic, in its entirety, no you haven't.

I am being perfectly literal.

Paths go backward. If something is called, it means a path was found back to it. If it finds a path back to void, nothing goes back from void, it has no predecessor. Nothing is called.

Void->... is the end of a path-branch, in failure.

I reflect upon (Top->A) = Void. If I have seen something which stably depends upon Top, no I haven't. Those don't exist.

I reflect upon (Unit->A) = Loop. If I have seen something which stably depends upon the unit, it is always there. No, my sibling cannot be this. Coyote cannot be this. This is utterly infinitely reliable.

I reflect upon (Void->A) = Top. If I have seen something which depends on nothing, those cannot be called forth, because nothing exists as a prerequisite substance. They have no binding parameters, and thus no constructor.

I reflect upon (Loop->A) = Unit. I suspect this has something to do with fixed points. If I have seen something which depends on infinity, it is uniquely defined up to isomorphism? Because there's really only one way that happens and it's like, a cast from the infinite loop?

I notice that I am confused.

I learned this from the Rationalists. They seem very confused.

There is a tendency I have. It is something like path-synesthesia.

It expresses the path-synesthesia in these symbols.

\x -> x

An object is inserted as an argument and then used once. This ensures it is the same object with logical certainty, because there could only be one path from the beginning to the end.

\x,y -> (x,y)

This is the same as \x -> \y -> (x,y). Any finite tuple is the same as a sequence of ordinary lambda functions. This is called Currying, after the same guy as Haskell. A unique path for any object there is preserved, including "the rest of the list."

\x^n -> repeat(n, \i -> (x,i))

An object is inserted as an argument and then used n times. This requires the object to have the property that it can be duplicated finitely many times.

\x^# -> infinite(\i -> (x,i))

infinite is the analogue of repeat applied with an infinity where a finite natural number should go. Therefore, it composes a function with itself infinitely many times. This requires a property to be able to be duplicated an infinite number of times. Possibly, one can also use the identity \x^# -> save(x) : (#A)->(#A).

\x;_ -> x, \_;y -> y

The selection of the first and second object from alternatives. The way such objects are arranged is with one resource and two paths forward. The path is taken when this selection of the alternative is reached. This is where the choice happens, when one option is replaced with _. The thing one passes in is two alternatives.

Void

It contains no objects except that generated by fix(\x->x). With the witness rule, the lambda has no witness, because x has the bottom type, which contains no clear example. We certainly cannot provide a witness to the void type, so it contains no objects and never will.

Loop

It contains no objects except that generated by fix(\x->x). We can't provide a witness to this type either.

Set A

A set contains:

1. nil : Set(Void) -> Set(A)

2. el : (A) -> Set(A)

3. and : (Set(A), Set(A)) -> Set(A)

This defines the empty set, points, and finite intersections.

If it were to contain infinite intersections, it would want to contain Set (Set A), the set of all sets of A. But when does this end? Is there a set of all sets of all sets? A -> Set A is a possible transformation, so it seems like Set (Set (Set (Set ... exists. Set(A) -> Set(Set(A)).
